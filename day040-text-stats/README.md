# Day 040: ファイル一括 文字数・行数統計ツール（Python）

フォルダの中のテキストファイルごとに、行数・文字数・単語数を数えて一覧表示する実務ツールです（外部ライブラリなし・ネット通信なし）。
原稿やコードの分量を、まとめて把握したいときに使えます。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python text_stats.py`（WindowsならダブルクリックでもOK）
2. 調べたいフォルダのパスを入れる
3. 対象の拡張子（空Enterで `.txt,.md`）とサブフォルダの有無を選ぶ
4. ファイルごとの統計と合計が表示される
5. 希望すれば `文字数統計.csv` に書き出し（Excelで開きやすいUTF-8・BOM付き）

## 機能
- ファイルごとの 行数 / 文字数 / 文字数(空白除く) / 単語数(空白区切り) を集計
- 全ファイルの合計も表示
- 文字コードは UTF-8 / Shift_JIS(cp932) を自動判別
- 大きすぎるファイル（20MB超）やバイナリ（ヌル文字を含む）は対象外
- 「.」で始まる隠しフォルダの中は対象外、自分自身も除外
- CSV出力はファイル名の数式インジェクションを無害化、連番で**上書きしない**

## 使った技術
- Python（標準ライブラリ `os` / `csv` / `sys` / `pathlib` のみ・外部通信なし）

## 学び・ポイント
- 行数は `splitlines()` の数。空ファイルは0行、`"a\n"` は1行、`"a\nb"` は2行
- 「単語数」は空白で区切ったかたまりの数。日本語は空白が無いと1かたまりになるので、
  英語の文章向けの目安と考える（表示名も「単語数(空白区切り)」にした）
- CSVにファイル名を書くときは、先頭に空白を入れた `  =cmd` のような回避も防ぐため、
  **空白を除いた先頭文字**も見て無害化する
- バイナリ判定はヌル文字で見分けるが、これは完全ではない（拡張子で絞るのが基本）

## 注意
- このツールはファイルを**書き換えません**（数えて表示・CSV出力のみ）。
- 「単語数」は空白区切りの目安です（日本語の語数とは異なります）。
