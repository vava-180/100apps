# Day 021: ファイル一括リネームツール（Python）

フォルダの中のファイル名を、まとめて変更する実務ツールです。フェーズ2（Python）の1本目。
変更前に必ずプレビューし、確認してから実行します（外部ライブラリなし・ネット通信なし）。

## 必要なもの
- Python（3.8以上の目安）。`python --version` で確認できます。

## 使い方
1. ターミナルでこのフォルダに移動し、`python rename.py` を実行
   （Windowsなら `rename.py` をダブルクリックでもOK）
2. 対象フォルダのパスを入れる（未入力ならこのスクリプトと同じ場所）
3. 変更方法を選ぶ
   - 連番をつける（例: file_001.jpg, file_002.jpg …）
   - 文字を置きかえる（名前の中の文字を別の文字に）
   - 前後に文字を足す（接頭辞・接尾辞）
4. 「変更前 → 変更後」のプレビューを確認
5. `yes` と入力したときだけ、実際に変更される

## 機能
- 3つのリネーム方法（連番／置換／接頭辞・接尾辞）
- 実行前のプレビューと最終確認（誤操作を防ぐ）
- 名前の衝突（上書きの危険）を検知して中止
- Windowsで使えない名前（禁止文字・予約語・末尾の空白など）を検知して中止
- A→B, B→C のような連鎖や、A↔B の入れ替えも安全（2段階リネーム）

## 使った技術
- Python（標準ライブラリ `pathlib` / `re` / `uuid` のみ・外部通信なし）

## 学び・ポイント
- ファイルを実際に変える前に、必ず**プレビューと確認**を挟むと事故を防げる
- 一括リネームの落とし穴は「最終的な名前は重複しないのに、変更の途中で名前がぶつかる」こと。
  いったん**全部を一時的な名前に退避してから最終名にする「2段階リネーム」**で安全にできる
- Windowsは大文字小文字を区別しないので、`file.txt→FILE.txt` のような変更も
  単純な方法だと「もうある」と誤判定されやすい。2段階方式ならこれも正しく変更できる
- 失敗したときに**できる限り元へ戻す（ロールバック）**ことで、中途半端な状態を避ける

## 注意
- 対象はフォルダ直下のファイルのみ（サブフォルダ・隠しファイルは対象外）。
- 拡張子は最後の1つだけを保持します（`archive.tar.gz` は `.gz` を拡張子として扱います）。
