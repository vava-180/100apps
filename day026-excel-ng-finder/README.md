# Day 026: Excel「NG箇所」検出＆一覧化ツール（Python）

Excelで作ったチェックリストなどから、指定した文字（例: `NG`）を含むセルを
**全シートまとめて**探し出し、新しいシート「検出結果」に一覧化する実務ツールです。
一覧の各行には**元のセルへ飛ぶリンク**が付くので、クリックでその箇所へジャンプできます。

## 必要なもの
- Python（3.8以上の目安）
- `openpyxl`（Excelを読み書きするライブラリ）。1回だけ次でインストール:
  ```
  pip install openpyxl
  ```
  ※ `openpyxl` はネット通信をするものではなく、Excelをローカルで処理するためのライブラリです。

## 使い方
1. ターミナルで `python ng_finder.py`（WindowsならダブルクリックでもOK）
2. 調べたいExcelファイル（.xlsx）のパスを入れる
3. 探す文字（例: `NG`）を入れる
4. 一致のしかた（含む／ぴったり同じ）と、大文字小文字を区別するかを選ぶ
5. 元ファイルと同じ場所に `〇〇_検出結果.xlsx` ができる（先頭に「検出結果」シート）

## 機能
- 全シート・全セルを走査して、指定文字を含む（または完全一致する）セルを検出
- 新シート「検出結果」に「シート名・セル番地・値」を一覧化
- セル番地は**元のセルへ飛ぶリンク**（同じブック内ジャンプ）付き
- 元ファイルは上書きせず、別名で保存（既に同名があれば連番）

## 使った技術
- Python ＋ `openpyxl`（外部通信なし）

## 学び・ポイント
- 数式セルは「式の本文」と「計算結果」がちがう。`=IF(...,"NG","OK")` のような式は、
  本文に `NG` の文字が入っているので、本文を検索すると**誤検出**する。
  そこで検索は `data_only=True`（**計算結果**を読むモード）で開いて判定している
- 同じブック内のセルへ飛ぶリンクは `=HYPERLINK("#'シート名'!B5", ...)` で作れる。
  シート名に含まれる `'` は `''` に2重化しないと式が壊れる
- 値が `=NG` のように `=` で始まると、書き込み先で数式と誤解されるので、
  セルを**文字列型に固定**して防いでいる

## 注意
- このツールは **.xlsx 専用**です（古い `.xls` は未対応。Excelで `.xlsx` に保存し直してください）。
- 数式セルの「計算結果」での検索は、**一度Excelで開いて保存したファイル**で有効です。
  プログラム等で作っただけで一度もExcelで保存していないファイルは、計算結果が記録されておらず
  数式セルが検索対象外になることがあります（手入力した文字は常に検索できます）。
- 結合セルは左上のセルだけが値を持つため、検出されるのは結合範囲の左上セルです。
- 過去にこのツールが作った「検出結果」シートは、検索対象から自動で除外します。
