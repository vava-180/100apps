# Day 061: スライド内容をMarkdownに書き出し（PowerPoint VBA）

開いているPowerPointの各スライドを巡り、**タイトル・本文・表**をMarkdownテキストに変換して、プレゼンと同じフォルダに `.md` ファイルとして保存するマクロです。資料の内容をテキストとして再利用（議事録・ブログ・ドキュメント化）したいときに便利です。

## 必要なもの
- Microsoft PowerPoint（VBAが使えるデスクトップ版）

## 使い方
1. 書き出したいプレゼンを開き、**一度保存**しておく（保存先フォルダに `.md` を作るため）
2. `Alt + F11` でVBE →「ファイル → ファイルのインポート」で `ExportSlidesToMarkdown.bas` を取り込む
3. `Alt + F8` →「ExportSlidesToMarkdown」を実行
4. 同じフォルダに「（プレゼン名）.md」ができる

## 書き出す内容
- スライドのタイトル → 見出し（`## スライドN: タイトル`）
- 本文の各段落 → 箇条書き（`- 文章`）。インデントの深さを段差（ネスト）に反映
- 表（テーブル）→ Markdownの表（1行目をヘッダ扱い）

## 対象外
- ノート（発表者メモ）、SmartArt、画像内の文字、グラフ

## 使った技術
- PowerPoint VBA（Slides / Shapes / TextRange.Paragraphs / Table）
- ADODB.Stream（UTF-8でテキスト保存）

## 学び・ポイント
- 段落ごとの階層は `TextRange.IndentLevel`（1〜5）で取れる。これを使ってMarkdownの箇条書きの段差を作れる
- スライドのタイトルは `Slide.Shapes.HasTitle` / `Slide.Shapes.Title` で簡単に取れる。本文側ではタイトルのプレースホルダを飛ばして二重に書き出さないようにする
- VBAの `Print #` はANSI（cp932）で書くため、日本語の `.md` が環境によって化けることがある。**`ADODB.Stream` の Charset="UTF-8" で保存**すれば、どこで開いても読める
- 表のセル内の改行や `|` はMarkdownの表を壊すので、スペースや `\|` に置き換えてから書き出す

## 注意
- 同名の `.md` があると上書きします。
- 文字や図形の「見た目の順番」ではなく、図形の並び順で書き出します（必要なら出力後に並べ替えてください）。
- マクロを含むブックは `.pptm` 形式で保存してください。
