# Day 030: CSV集計・ピボットツール（Python）

1つのCSVを読み込み、選んだ列でまとめて**集計**する実務ツールです（外部ライブラリなし・ネット通信なし）。
「担当者ごとの金額合計」のような単純集計だけでなく、「担当者 × 月」のような**クロス集計（ピボット表）**も作れます。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python aggregate.py`（WindowsならダブルクリックでもOK）
2. 集計したいCSVファイルのパスを入れる
3. 番号付きで列が一覧表示されるので、
   - 「行の見出しにする列」を番号で選ぶ
   - 「列の見出しにする列」を選ぶ（ピボットしないなら空Enter）
   - 集計方法（合計／件数／平均／最大／最小）を選ぶ
   - 件数以外なら「集計する値の列」を選ぶ
4. 集計結果のプレビューを確認
5. `yes` で `元の名前_集計.csv` に書き出し（Excelで開きやすいUTF-8・BOM付き）

## 機能
- 5種類の集計（合計／件数／平均／最大／最小）
- 任意でクロス集計（ピボット表）にできる
- 桁区切りカンマ（`1,234`）も数値として読む（`1,2` のような不正な形は数値扱いしない）
- 文字コードは UTF-8 / Shift_JIS(cp932) を自動判別し、読めた方を画面に表示
- 見出しが無いCSVや、列が欠けた行は数えず、件数を分けて報告
- 出力は連番で**上書きしない**（元のCSVも書き換えない）

## 使った技術
- Python（標準ライブラリ `csv` のみ・外部通信なし）

## 学び・ポイント
- 集計は「`(行キー, 列キー)` をキーにした辞書」に数値をためると、単純集計もピボットも同じ仕組みで書ける
- 列は「名前」ではなく**番号で選んでもらう**と、同じ見出し名が複数あっても迷わない
- 桁区切りカンマは便利だが、`1,2` まで通すと `12` と誤読する。**正しい区切りの形か確認してから**取り除く
- 平均は小数2桁、合計・最大・最小は精度を落とさない、と**集計ごとに表示ルールを分ける**と実用的
- 出力は `"x"`（新規作成専用）で開き、既にあれば連番にやり直すと、**既存ファイルを上書きしない**

## 注意
- 数値として読めない値（空欄・文字など）は、その値だけ集計から外します（件数には数えます）。
- 行/列の見出しにした列そのものが無い（列数が足りない）行は、誤集計を防ぐため集計から外します。
