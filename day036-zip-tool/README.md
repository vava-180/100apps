# Day 036: ZIP一括 圧縮・解凍ツール（Python）

フォルダの中のファイルを1つのZIPにまとめたり、フォルダの中にある複数のZIPをまとめて解凍したりする実務ツールです（外部ライブラリなし・ネット通信なし）。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python zip_tool.py`（WindowsならダブルクリックでもOK）
2. やることを選ぶ（1=圧縮／2=解凍）
3. 圧縮：フォルダを指定 → サブフォルダを含めるか選ぶ → ZIP名を決める
4. 解凍：ZIPが入ったフォルダを指定 → 各ZIPが「ZIP名のフォルダ」に展開される

## 機能
- 圧縮：フォルダ内のファイルを1つのZIPにまとめる（サブフォルダの構造も保持）
- 解凍：フォルダ内の各ZIPを、それぞれ専用フォルダに展開
- **Zip Slip 対策**：ZIP内に `..` や絶対パスでフォルダの外へ書き出す細工があっても、指定フォルダの外には書き出さない
- ZIP内で同じ場所に展開される重複（同名・大文字小文字違い）は、内部上書きを防ぐため除外
- 圧縮時はショートカット/シンボリックリンクを安全のため対象外
- 壊れたZIP・パスワード付きZIPは飛ばし、作りかけの展開先は片付ける
- 出力ZIP・展開先フォルダは連番で**上書きしない**

## 使った技術
- Python（標準ライブラリ `zipfile` / `shutil` / `os` / `sys` / `pathlib` のみ・外部通信なし）

## 学び・ポイント
- ZIPの解凍は `extractall()` を使うと「Zip Slip」（ZIP内の `..` で外へ書き出す攻撃）に弱い。
  **1件ずつ展開先を検算**（`resolve()` してフォルダ内か確認）してから書くと安全
- ZIPの中に同名や大文字小文字違いの項目があると、後の項目が前を**上書き**する。
  展開先をキーにして**重複を除外**すると、意図しない上書きを防げる
- 圧縮で「フォルダ内のリンク」を入れると、リンク先（外の実体）まで取り込んでしまう。
  リンクは対象外にすると安全
- ZIPは `"x"`（新規作成専用）で開き、展開先フォルダも連番にして、上書き事故を防ぐ

## 注意
- パスワード付きZIPの解凍には対応していません（飛ばします）。
- 圧縮対象にショートカット/シンボリックリンクは含めません。
