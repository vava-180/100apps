# Day 032: フォルダ容量レポートツール（Python）

指定したフォルダの直下にある「サブフォルダ」「ファイル」が、それぞれどれくらいの容量かを調べ、**大きい順**に一覧表示する実務ツールです（外部ライブラリなし・ネット通信なし）。
「どこが容量を食っているか」を探す、ディスク整理の前段に向いています。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python folder_size.py`（WindowsならダブルクリックでもOK）
2. 調べたいフォルダのパスを入れる（未入力ならこのスクリプトと同じ場所）
3. 大きい順に容量が一覧表示される
4. 希望すれば `容量レポート.csv` に書き出し（Excelで開きやすいUTF-8・BOM付き）

## 機能
- 直下の各サブフォルダ／ファイルの容量を**大きい順**に表示
- サブフォルダは中身をすべてたどって合計
- 容量は読みやすい単位（KB / MB / GB …、1024ごと）に変換
- シンボリックリンクはたどらない（無限ループ・二重計算を防止）
- 権限エラーや走査中に消えたファイルは、その分だけ飛ばして件数を表示
- 任意でCSVレポート出力（連番で上書きしない）

## 使った技術
- Python（標準ライブラリ `os` / `pathlib` / `csv` のみ・外部通信なし）

## 学び・ポイント
- フォルダの合計サイズは `os.walk` で中身を全部たどり、各ファイルのサイズを足す
- シンボリックリンク（別の場所への近道）は**たどると無限ループや二重計算**になりやすい。
  `followlinks=False` と `os.stat(follow_symlinks=False)` で「リンク自身」だけを見る
- リンク判定は**先に確認**する。リンクに対して `is_dir()` を呼ぶとリンク先を見に行ってしまう
- 容量レポートは「移動・削除はせず、見るだけ」にすると安全に使える

## 注意
- このツールはファイルを**移動・削除しません**（容量を調べて表示するだけ）。
- 容量の単位は1024基準（1KB = 1024バイト）で計算しています。
