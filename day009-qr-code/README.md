# Day 009: QRコード生成

URLや文章を入力すると、その場でQRコードを作るブラウザツールです。**外部サービスを一切使わず**、QRの計算もブラウザの中だけで完結します（ネット通信なし）。

## 使い方
1. `index.html` をブラウザで開く
2. URLやテキストを入力する（入力するたびに自動でQRが更新される）
3. スライダーでサイズを調整する
4. 「PNGで保存」で画像を保存、「画像をコピー」でクリップボードにコピー
5. スマホのカメラやQRアプリで読み取って確認する

## 機能
- 入力に合わせてリアルタイムでQR生成
- バージョン1〜6を自動選択（最大106バイト／日本語は1文字3バイトの目安）
- 誤り訂正レベルM
- 表示サイズの調整
- PNG保存・画像コピー

## 使った技術
- HTML / CSS / JavaScript（1ファイル完結・**外部通信なし**）
- QR生成のアルゴリズムをすべて自前で実装（ガロア体GF(256)、リード・ソロモン符号、マスク選択など）
  - 参考：Nayuki氏のQR Code generatorのアルゴリズム

## 開発用テスト（任意）
QRが正しく作れているかを自動確認するスクリプトを同梱しています（アプリ本体とは別物）。

```
node qr-self-test.js
```

- 生成したQRを**独立して書いたデコーダで読み戻し**、元の文字列に一致するか（日本語含む）
- 誤り訂正符号（リード・ソロモン）が数学的に正しいか
- バージョン自動選択の境界・フォーマット情報が正しいか

を確認します（Node.jsのみ・外部ライブラリ/通信なし）。

## 学び・ポイント
- QRコードは「データ＋誤り訂正コード」を特殊な計算（リード・ソロモン符号）で作り、
  決まった順番でマス目に並べ、8種類のマスクから一番読みやすいものを選んで作られている
- 外部APIを使わなくても、計算ライブラリを同梱（今回は自作）すればQRは作れる

## 注意
- 対応はバージョン1〜6・レベルM（最大約106バイト）。長い文章はエラー表示になります。
- 文字はUTF-8（バイトモード）で入れています。**URLや英数字は問題なく読めます**が、
  日本語・絵文字は「文字コードの宣言（ECIモード26）」を省いているため、
  一部の古いQRリーダーで文字化けする可能性があります（多くの最新スマホでは読めます）。
- 生成後はスマホで一度読み取って確認することをおすすめします。
