# Day 029: 文字コード一括変換ツール（Python）

フォルダの中の複数テキストファイルの「文字コード」を、まとめて変換する実務ツールです（外部ライブラリなし・ネット通信なし）。
日本語環境では UTF-8 と Shift_JIS が混在しがちで文字化けの原因になります。これで文字コードをそろえられます。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python convert_encoding.py`（WindowsならダブルクリックでもOK）
2. 変換したいフォルダのパスを入れる
3. 変換先を選ぶ
   - 1) UTF-8（BOM無し）
   - 2) UTF-8（BOM付き・Excel向き）
   - 3) Shift_JIS（cp932）
4. 対象の拡張子・サブフォルダの有無を選ぶ
5. プレビューを確認し、控え(.bak)を残すか選び、`yes` で実行

## 機能
- 文字コードの一括変換（UTF-8 BOM無し／BOM付き／Shift_JIS）
- 今の文字コードを自動判別（BOM付きUTF-8 / UTF-8 / Shift_JIS）
- すでに変換先と同じ形のファイルは、そのまま（無駄に書き換えない）
- 変換先で表せない文字（絵文字など）を含むファイルは、データを壊さないよう丸ごとスキップ
- 任意で変換前の控え（.bak）を保存（同名があれば連番）

## 使った技術
- Python（標準ライブラリ `shutil` / `re` のみ・外部通信なし）

## 学び・ポイント
- 「BOM」はファイル先頭に付く小さな目印。Excelでの文字化け対策にBOM付きUTF-8が使われる
- 「変換後のバイト列が元と同じ」なら、すでにその形なので**書き換えない**（ASCIIだけのファイルなどが該当）
- Shift_JIS は表せない文字があると保存時に欠けてしまうので、`errors`を既定（厳しい）にして
  **表せない時はその1ファイルを飛ばす**（一部だけ欠けた中途半端な保存を避ける）
- Shift_JIS はほぼどんなバイトも読めてしまうため、ヌル文字を含むバイナリは対象外にする

## 注意
- 控え(.bak)を残さない設定だと、元に戻せません。心配なときは控えを残してください。
- Shift_JIS への変換は、もとに絵文字や一部の特殊文字があると、そのファイルは変換されません（一覧に表示します）。
