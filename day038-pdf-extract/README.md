# Day 038: PDFテキスト抽出ツール（Python）

PDFファイルから文字（テキスト）を取り出して、`.txt` ファイルに保存する実務ツールです。
1つのPDFでも、フォルダの中の複数PDFでもまとめて処理できます（ローカル動作・ネット通信なし）。

## 必要なもの
- Python（3.8以上の目安）
- `pypdf`（PDFを読むライブラリ）。1回だけ次でインストール:
  ```
  python -m pip install pypdf
  ```
  ※ `pypdf` はネット通信をするものではなく、PDFをローカルで処理するためのライブラリです。

## 使い方
1. ターミナルで `python extract_text.py`（WindowsならダブルクリックでもOK）
2. PDFファイル、またはPDFが入ったフォルダのパスを入れる
3. 各PDFの文字が取り出され、`元の名前.txt` として保存される

## 機能
- 1つのPDF／フォルダ内の複数PDFをまとめてテキスト抽出
- ページごとに区切り（`----- ページ 1 -----`）を入れて保存
- パスワード付きPDF・壊れたPDFは飛ばす（理由を表示）
- 1ページだけ壊れていても、そのページを飛ばして残りを取り出す
- 文字が取り出せないPDF（スキャン画像など）は、**空のtxtを作らず**スキップ
- 出力は連番で**上書きしない**（元のPDFは読むだけ）

## 使った技術
- Python ＋ `pypdf`（外部通信なし）

## 学び・ポイント
- PDF操作は標準ライブラリだけでは難しいので、専用ライブラリ `pypdf` を使う
  （ネット通信をしないライブラリは、同梱して使ってよい）
- 画像になっている文字（スキャンPDF）は `extract_text()` では取り出せない。
  「**実際に文字が取れたページがあるか**」を見て、無ければtxtを作らないようにする
  （そうしないと、ページ見出しだけの空txtができてしまう）
- 1ページごとに `try` で囲むと、1ページ壊れていてもPDF全体が失敗にならない
- パスワード付きPDFは `is_encrypted` で見分け、空パスワードで開けなければ飛ばす

## 注意
- フォルダ指定のときは、その直下のPDFだけが対象です（サブフォルダの中は見ません）。
- スキャン画像のPDFは、文字情報が無いため取り出せません。
