# Day 042: 複数ブック集約マクロ（Excel VBA）

指定フォルダの中にある複数のExcel（.xlsx）の「1枚目シート」のデータを、**1枚のシートに縦に積み上げて**集約するマクロです。各行には「元ファイル」名が付きます。
（Day 039 のPython版は「ファイルごとに別シート」でしたが、こちらは「全部を1シートに縦結合」する別アプローチです）

## 必要なもの
- Microsoft Excel（VBAが使えるデスクトップ版）

## 使い方
1. `Alt + F11` でVBE → 「ファイル → ファイルのインポート」で `ConsolidateWorkbooks.bas` を取り込む
   （または中身をコピーして標準モジュールに貼り付け）
2. `Alt + F8` →「ConsolidateWorkbooks」を実行
3. 集約したいExcelが入ったフォルダのパスを入力（例: `sample` フォルダ）
4. 確認で「はい」→ 新しいブックに集約結果が出力される（保存はご自身で）

## 機能
- フォルダ内の複数 .xlsx を1シートに縦結合（先頭に「元ファイル」列）
- ファイルを**名前順**に処理し、最初の中身のあるファイルの見出しを採用
- 元ブックは**読み取り専用**で開き、保存せず閉じる（中身は変えない）
- 1ファイル／1シートが壊れていても、そこだけ失敗として数え、残りを継続
- 中身が空のファイル・列数がちがうファイルは件数で報告
- 出力行が上限（約104万行）に達したら打ち切って知らせる

## 使った技術
- Excel VBA（Workbooks.Open / Range.Find / Collection・挿入ソート）

## 学び・ポイント
- 最終行は**A列だけで判断しない**。`Cells.Find(What:="*", SearchDirection:=xlPrevious)` で
  シート全体から探すと、A列が空でも他の列にデータがある行を落とさない
- `Dir$` のループ中に `Workbooks.Open` などを混ぜると不安定になりやすいので、
  **先にファイル名を集めてから**開く
- 大量データでは出力行の上限（1,048,576行）に注意。**書く前に上限チェック**する
- 1ファイルの処理を別関数にして、失敗してもそのファイルだけ飛ばして続けると、集約ツールらしくなる

## 注意
- 各ファイルの**列の並びは同じ前提**です（最初のファイルの見出しの列数ぶんを読みます）。
- 結果は新しいブックに出ます。必要に応じて `.xlsx` などで保存してください。
- マクロを含むブックは `.xlsm` 形式で保存してください。

## サンプル
- `sample/支店A・B・C.xlsx` … 同じ列（商品・数量・金額）の売上データ。
  `sample` フォルダを指定すると、3ファイル・計5行が「元ファイル」列付きで1枚にまとまります。
