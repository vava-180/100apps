# Day 093: 画像加工ツール（Python + Pillow）

フォルダの中の画像（または1枚）を、まとめて加工するコマンドラインツールです。元画像は上書きせず、別フォルダ `kakou_出力` に保存します。

## 必要なもの
- Python 3
- Pillow（画像ライブラリ）… 未導入なら `pip install pillow`

## 使い方
1. ターミナルでこのフォルダに移動する
2. 次のどちらかで起動する
   - 対話形式：`python image_editor.py`
   - パス指定：`python image_editor.py 画像フォルダ`（例: `python image_editor.py sample`）
3. 加工メニューを選ぶ
   - `1` グレースケール（白黒）
   - `2` 幅を指定してリサイズ（縦横比はそのまま）
   - `3` 時計回りに回転（90 / 180 / 270 度）
4. `kakou_出力` フォルダに加工後の画像ができる

## 機能
- 対応形式：jpg / jpeg / png / gif / bmp / webp（フォルダ直下のみ）
- グレースケール・リサイズ・回転（時計回り）の3種
- 元画像は上書きしない（`kakou_出力` に同名で保存）
- 1枚が壊れていても、残りの処理は続ける（成功/失敗の枚数を表示）
- 入力チェック（メニュー・幅・角度は正しく入れ直すまで聞き直す）
- Windowsの引用符付きパス（`"C:\...\img.jpg"`）にも対応

## 使った技術
- Python（pathlib / Pillow）

## 学び・ポイント
- リサイズで高さが0にならないよう `max(1, round(h * width / w))` で下限を1にする
- Pillow の `Transpose.ROTATE_90` は「反時計回り90度」なので、時計回りに合わせるには 90↔270 を入れ替える
- JPEGは透明や白黒(L)をそのまま保存できないことがあるので、保存前に `convert("RGB")` する
- Windowsのコンソール（cp932）では `✓` などが表示できず落ちる。記号はASCIIにし、出力は `encode(errors="replace")` で守る

## 注意
- 完全にローカルで動きます（外部APIやネット通信はありません）。
- 同名の過去の出力は上書きされます。

## サンプル
- `sample/` … テスト用のPNG画像3枚（photo1〜3.png）。`python image_editor.py sample` で試せます。
