# Day 027: テキスト一括置換ツール（Python）

フォルダの中の複数テキストファイルから、指定した文字をまとめて別の文字に置き換える実務ツールです（外部ライブラリなし・ネット通信なし）。
いきなり書き換えず、まず「どのファイルで何件置き換わるか」をプレビューし、`yes` で実行します。

## 必要なもの
- Python（3.8以上の目安）

## 使い方
1. ターミナルで `python replace_text.py`（WindowsならダブルクリックでもOK）
2. 置換したいフォルダのパスを入れる
3. 「探す文字」と「置き換える文字」を入れる（置き換える文字は空Enterで削除）
4. 対象の拡張子・サブフォルダの有無・大文字小文字の区別を選ぶ
5. プレビュー（件数）を確認し、控え(.bak)を残すか選び、`yes` で実行

## 機能
- 複数テキストファイルの一括置換（プレビュー → 確認 → 実行）
- 置換件数を事前に表示。1件も無いファイルは触らない
- 文字コードは BOM付きUTF-8 / BOM無しUTF-8 / Shift_JIS(cp932) を自動判別し、**元の形を保って書き戻す**
- 任意で置換前の控え（.bak）を保存（同名があれば連番）

## 使った技術
- Python（標準ライブラリ `re` / `shutil` のみ・外部通信なし）

## 学び・ポイント
- 探す文字に `.` や `(` などの記号が来てもいいように、`re.escape()` で「ただの文字」として扱う
- BOM無しUTF-8のファイルを読んで書き戻すとき、うっかりBOMを足さないよう、
  先頭バイトを見て**元の形に合わせる**（読み書きで形が変わると別ツールで文字化けの原因になる）
- 文字コードを保ち、改行も変えないよう、書き込みは `encode()` してから**バイトで書く**
- cp932のファイルに、cp932で表せない文字（絵文字など）を入れようとすると保存時に失敗するので、
  **プレビュー段階で保存できるか試して**、できないファイルは飛ばす

## 注意
- 控え(.bak)を残さない設定だと、元に戻せません。心配なときは控えを残してください。
- プレビュー後にファイルが外部で変更され、置換件数が変わった場合は、安全のためそのファイルを飛ばします。
