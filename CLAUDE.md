# CLAUDE.md

このファイルは Claude Code が毎回最初に読み込む、プロジェクトの説明書です。

## このプロジェクト
- 100日間で100個のアプリを作る個人チャレンジ「100apps」
- 作者はIT初心者。Claudeはプロのプログラマーとしてサポートする
- 成果物はGitHubで公開し、noteで記録する

## サポート方針
- コードも説明も「初心者にわかる言葉」で。専門用語は一言かみ砕く
- 何を作る／変えるかを一言説明してから実装する
- コードには日本語コメントを付ける

## フォルダ構成
- 1アプリ = 1フォルダ。`dayXXX-アプリ名/`（例: `day001-character-counter/`）
- 各フォルダに アプリ本体（例: index.html）と README.md を置く
- README.md は `day-README-template.md` の形式に合わせる

## 技術のデフォルト（フェーズ別）
- Day 1–20: ブラウザツール → HTML / CSS / JavaScript（1ファイル完結が基本）
- Day 21–40: 実務ツール → Python
- Day 41–65: Office自動化 → VBA（Excel / PowerPoint / Outlook）
- Day 66–90: スマホアプリ → Flutter
- 上記以外は、その用途に最適な言語をその都度選ぶ

## 毎日のワークフロー
1. 仕様メモ（何をする／完成の条件を1〜2行）
2. 構想して実装する
3. **コード実装後は必ずCodexで厳しめレビューを受ける**（動くか・読みやすいか・エラー処理）
4. 指摘があれば必ず反映（修正）する
5. フォルダのREADME作成＋スクショ＋ルートREADMEの状態を更新
6. コミットしてプッシュする（メッセージ例: `Day 001: 文字数カウンター`）
   - **push前に、Codexチェックが完了していることを必ず確認する**

## コードレビューの必須ルール（IMPORTANT）
- **IMPORTANT: コードを実装したら、必ずCodexで「厳しめ」のレビューを行う。**
- **IMPORTANT: レビューで指摘があれば、必ず反映（修正）してから先に進む。**
- **IMPORTANT: git push の前に、Codexチェックが完了していることを必ず確認する。** 未完了のままpushしない。

詳しいルールは別ファイルにまとめてある。必ず守ること。
