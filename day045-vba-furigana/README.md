# Day 045: ふりがな自動付与（Excel VBA）

選んだ1列（氏名など）の読み（ふりがな）を求めて、すぐ右の列に書き出すマクロです。カタカナ／ひらがなを選べます。

## 必要なもの
- Microsoft Excel（VBAが使えるデスクトップ版・日本語IME環境）

## 使い方
1. 氏名などが入った「1列分」を選ぶ（`sample/ふりがなサンプル.xlsx` でも試せます）
2. `Alt + F11` でVBE →「ファイル → ファイルのインポート」で `AddFurigana.bas` を取り込む
3. `Alt + F8` →「AddFurigana」を実行
4. `1`（カタカナ）か `2`（ひらがな）を選ぶ
5. 右の列にふりがなが入る

## 機能
- `Application.GetPhonetic` で漢字の読みを取得し、右の列に書き出し
- カタカナ／ひらがなを選べる（ひらがなは `StrConv(vbHiragana)` で変換）
- 右の列に既存値があれば、対象範囲を示して上書き確認
- 1列・ひと続きの選択に限定（最右列は不可）
- 空セル・エラー値セルは飛ばし、読みを取得できなかったセルも件数で報告

## 使った技術
- Excel VBA（Application.GetPhonetic / StrConv / Range.Offset / Intersect）

## 学び・ポイント
- 読みは `Application.GetPhonetic(文字列)` で取れる（ExcelのIME辞書を使う）
- 右の列に書くときは、最右列(XFD)だと `Offset(0,1)` が無くてエラーになるので、**事前にチェック**する
- 1セルずつ `On Error Resume Next` で囲むと、特定セルで読みが取れなくても全体が止まらない
- 「自動推定」なので、人名などは読み違いが起きる。完了メッセージでも**要確認**を促す

## 注意
- 読みはExcel/IMEの辞書による**推定**です。人名などは読み違いが出ることがあるので、必ず確認してください。
- 英数字・記号は期待した読みにならない場合があります。
- ふりがなは「右の列」に書きます。右の列の内容は上書きされることがあります（確認あり）。
- マクロを含むブックは `.xlsm` 形式で保存してください。

## サンプル
- `sample/ふりがなサンプル.xlsx` … A列に氏名、B列（ふりがな）は空。
  A列を選んで実行すると、B列に読みが入ります。
