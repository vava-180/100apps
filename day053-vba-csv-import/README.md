# Day 053: CSV取り込み整形マクロ（Excel VBA）

CSVファイルを新しいシートに取り込み、見やすく整えるマクロです。
Excelの「CSVを開く」だと崩れてしまう**先頭ゼロ（電話番号・社員番号など）**を、すべて文字列として取り込んで保ちます。

## 必要なもの
- Microsoft Excel（VBAが使えるデスクトップ版・Windows）

## 使い方
1. `Alt + F11` でVBE →「ファイル → ファイルのインポート」で `ImportCsv.bas` を取り込む
2. `Alt + F8` →「ImportCsv」を実行
3. CSVファイルのパスを入力（`sample/取り込みサンプル.csv` で試せます）

## 機能
- すべて文字列として取り込み（先頭ゼロや日付の自動変換を防ぐ）
- 各セルの前後の空白をトリム
- 見出し（1行目）を太字・先頭行を固定・列幅を自動調整
- 文字コードは BOM で判定（UTF-8 / Shift-JIS）
- ダブルクォート対応のCSV解析（`"a,b"` や `""` のエスケープOK）
- 大きすぎるCSVや、作成途中の失敗時はシートを残さない

## 使った技術
- Excel VBA（ADODB.Stream で文字コード判定 / 2次元配列の一括書き込み）

## 学び・ポイント
- 先頭ゼロを残すには、書き込み前にセルを**文字列書式（`NumberFormat = "@"`）**にする
- 文字コードは先頭の**BOM**（UTF-8は `EF BB BF`）で見分けられる。ADODB.Streamは
  `Charset` を指定して読めるので、UTF-8とShift-JISの両対応がしやすい
- 1セルずつ書くと遅いので、**2次元配列に詰めてから `Range.Value` で一括**書き込みすると速い
- 引用符が閉じていない行はエラーにして、壊れたCSVを黙って取り込まない

## 注意
- 1セルの中に改行を含むCSV（セル内改行）は対応していません（1行=1レコード前提）。
- すべて文字列で取り込むため、数値として計算したい列は後で数値に直してください。
- 空行は取り込みません。各セルは前後の空白をトリムします。
- マクロを含むブックは `.xlsm` 形式で保存してください。

## サンプル
- `sample/取り込みサンプル.csv` … 先頭ゼロの社員番号・電話番号、クォート内カンマを含むデータ。
  取り込むと、先頭ゼロが消えずに文字列のまま入ります。
